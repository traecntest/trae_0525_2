# 用户与业务模块统一验证标准

## 1. 概述
本文档定义了用户管理模块和业务模块所有字段的统一验证规则，确保前端表单验证和后端 API 验证完全一致。

## 2. 用户管理模块验证规则

### 2.1 用户名 (username)

| 项目 | 规则 |
|------|------|
| 必填 | 是 |
| 最小长度 | 3 个字符 |
| 最大长度 | 50 个字符 |
| 格式 | 字母、数字、下划线、连字符，必须以字母开头 |
| 允许字符 | `a-z`, `A-Z`, `0-9`, `_`, `-` |
| 唯一性 | 同一租户下唯一 |
| 错误提示 | "用户名格式不正确，仅支持字母、数字、下划线和连字符，且必须以字母开头" |

### 2.2 密码 (password)

| 项目 | 规则 |
|------|------|
| 必填 | 创建时必填，更新时可选 |
| 最小长度 | 8 个字符 |
| 最大长度 | 100 个字符 |
| 强度要求 | 至少包含字母和数字，建议包含特殊字符 |
| 错误提示 | "密码长度至少 8 位，且需包含字母和数字" |

### 2.3 邮箱 (email)

| 项目 | 规则 |
|------|------|
| 必填 | 否（可为空或 null） |
| 最大长度 | 100 个字符 |
| 格式 | 标准邮箱格式 |
| 唯一性 | 同一租户下唯一 |
| 错误提示 | "请输入有效的邮箱地址" |

### 2.4 手机号 (phone)

| 项目 | 规则 |
|------|------|
| 必填 | 否 |
| 最大长度 | 20 个字符 |
| 格式 | 中国大陆手机号（11位，1开头）或国际号码（可选） |
| 允许格式 | `1[3-9]\d{9}`（中国大陆）或 `+[1-9]\d{1,14}`（国际） |
| 错误提示 | "请输入有效的手机号码" |

### 2.5 姓名 (fullName)

| 项目 | 规则 |
|------|------|
| 必填 | 是 |
| 最大长度 | 100 个字符 |
| 格式 | 任意字符 |
| 错误提示 | "姓名不能超过 100 个字符" |

### 2.6 状态 (status)

| 项目 | 规则 |
|------|------|
| 类型 | 整数 |
| 可选值 | 0 (禁用), 1 (启用) |
| 默认值 | 1 |
| 错误提示 | "状态只能是 0 (禁用) 或 1 (启用)" |

## 3. 事件管理模块验证规则

### 3.1 事件标题 (title)

| 项目 | 规则 |
|------|------|
| 必填 | 是 |
| 最大长度 | 200 个字符 |
| 错误提示 | "事件标题不能为空" |

### 3.2 事件类型 (eventType)

| 项目 | 规则 |
|------|------|
| 必填 | 是 |
| 可选值 | ALERT(告警), INCIDENT(事件), MAINTENANCE(维护), OPERATION(运维), OTHER(其他) |
| 错误提示 | "事件类型必须是以下值之一: ALERT, INCIDENT, MAINTENANCE, OPERATION, OTHER" |

### 3.3 事件级别 (severity)

| 项目 | 规则 |
|------|------|
| 必填 | 是 |
| 可选值 | INFO(信息), WARNING(警告), MINOR(一般), MAJOR(严重), CRITICAL(紧急) |
| 默认值 | INFO |
| 错误提示 | "事件级别必须是以下值之一: INFO, WARNING, MINOR, MAJOR, CRITICAL" |

### 3.4 事件时间 (eventTime)

| 项目 | 规则 |
|------|------|
| 必填 | 是 |
| 类型 | 日期时间 |
| 错误提示 | "事件时间不能为空" |

### 3.5 事件状态 (status)

| 项目 | 规则 |
|------|------|
| 可选值 | NEW(新建), ACKNOWLEDGED(已确认), IN_PROGRESS(处理中), RESOLVED(已解决), CLOSED(已关闭) |
| 默认值 | NEW |

## 6. 任务中心模块验证规则

### 6.1 任务名称 (name)

| 项目 | 规则 |
|------|------|
| 必填 | 是 |
| 错误提示 | "任务名称不能为空" |

### 6.2 任务类型 (taskType)

| 项目 | 规则 |
|------|------|
| 必填 | 是 |

|------|------|
| 类型 | 整数 |

### 6.4 任务状态 (status)

| 项目 | 规则 |
|------|------|
| 可选值 | QUEUED(排队中), PROCESSING(处理中), COMPLETED(已完成), FAILED(失败), CANCELLED(已取消) |

## 7. 错误提示信息统一

### 7.1 前端提示
- 表单验证失败时，在字段下方显示红色错误提示
- 使用与后端一致的错误文案

### 7.2 后端提示
- API 返回统一格式的错误响应
- 错误信息使用中文
- 多个字段错误时，返回所有错误

## 8. 验证流程

1. **前端先验证**：用户输入时实时或提交时验证
2. **后端再验证**：API 接收请求后再次验证所有规则
3. **唯一性检查**：在数据库层面检查唯一性约束
